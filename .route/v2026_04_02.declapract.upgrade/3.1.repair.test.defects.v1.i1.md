## defect #2: test:lint:cycles fails on node_modules

**what**: dpdm detects circular deps in node_modules, not src/

**how**: `npm run test:lint:cycles` exits 1

**why**: dpdm follows imports into node_modules when it scans `src/**/*.ts`. the `--exclude '^$'` pattern is intentionally a no-op — the practice WANTS to check node_modules. the circular deps were internal to external packages (helpful-errors, type-fns).

**fix**: upgraded packages that had circular dependencies:
- helpful-errors 1.7.2 → 1.7.3
- type-fns 1.21.0 → 1.21.2
- simple-log-methods 0.6.9 → 0.6.12
- domain-objects 0.31.9 → 0.31.11
- declapract 0.13.16 → 0.13.19
- test-fns 1.15.7 → 1.15.8

**classification**: package bug (now fixed upstream, resolved via upgrade)
