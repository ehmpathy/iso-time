# 3.2.reflect.test.defects — classification

## defect #1: depcheck flags `rhachet-brains-xai` as unused

### classification: **practice bug**

### evidence

the rhachet best-practice adds the dependency:
```
ehmpathy/declapract-typescript-ehmpathy:src/practices/rhachet/best-practice/package.json
"rhachet-brains-xai": "@declapract{check.minVersion('0.3.1')}",
```

but the lint best-practice depcheckrc only ignores anthropic, not xai:
```
ehmpathy/declapract-typescript-ehmpathy:src/practices/lint/best-practice/.depcheckrc.yml
- rhachet-brains-anthropic
```

### why practice bug (not repo quirk)

this is not special to iso-time. any repo that:
1. adopts the `rhachet` best practice (which adds `rhachet-brains-xai`)
2. adopts the `lint` best practice (which runs depcheck)

will hit this defect. the practice templates are inconsistent with each other.

### upstream fix required

add `rhachet-brains-xai` to `src/practices/lint/best-practice/.depcheckrc.yml` ignores list:
```yaml
ignores:
  - rhachet-brains-anthropic
  - rhachet-brains-xai  # absent from practice
```

## defect #2: test:lint:cycles fails on node_modules

### classification: **package bug** (helpful-errors, type-fns)

### evidence

the lint best-practice uses dpdm to detect circular dependencies:
```
ehmpathy/declapract-typescript-ehmpathy:src/practices/lint/best-practice/package.json
"test:lint:cycles": "dpdm --no-warning --no-tree --exit-code circular:1 --exclude '^$' 'src/**/*.ts'",
```

the `--exclude '^$'` is intentionally a no-op — the practice WANTS to check node_modules for cycles. dpdm correctly follows imports into node_modules and finds cycles internal to helpful-errors and type-fns.

### why package bug (not practice bug)

the practice behaves as intended. the defect is in the packages themselves:
- `helpful-errors` has internal circular dependencies
- `type-fns` has internal circular dependencies

these packages need to fix their internal cycles.

### upstream fix required

fix circular dependencies in:
- `ehmpathy/helpful-errors`
- `ehmpathy/type-fns`

### local status

**FIXED** — upgraded packages that had circular dependencies:
- helpful-errors 1.7.2 → 1.7.3
- type-fns 1.21.0 → 1.21.2
- simple-log-methods 0.6.9 → 0.6.12
- domain-objects 0.31.9 → 0.31.11
- declapract 0.13.16 → 0.13.19
- test-fns 1.15.7 → 1.15.8

test:lint:cycles now passes.
